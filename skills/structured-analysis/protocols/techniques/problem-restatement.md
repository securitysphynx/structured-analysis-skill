# Protocol: Problem Restatement (Lean)

> **Phase**: Launch (Lean) | **Effort**: 5 minutes | **Bias Mitigated**: Anchoring
> **Output Artifact**: Appended to `working/problem-framing.md`
> **Library**: 00-prime §4 (Extended: Lean SATs), 00-prime §7 (Agile Rigor), 04-agile-rigor-update

---

## Purpose

5-minute rigor check. Rewrite the question at least 3 different ways to break anchoring bias and prevent Type III Error. This is the Lean SAT version of Issue Redefinition for time-pressured situations.

---

## Execution

### 1. SETUP
- Read the problem statement from conversation context

### 2. PRIME
State: "Quick reframing check — 3 alternative ways to state this problem."

### 3. EXECUTE

1. Rewrite the question in **3 different ways**, using these shifts:
   - Shift focus: Actor to System
   - Shift focus: Threat to Vulnerability
   - Shift focus: "Why" to "How" (or reverse)
2. Identify which restatement opens the most new analytical paths
3. Select and state the working framing

### 4. ARTIFACT
If `working/problem-framing.md` exists, append a "Lean Restatement" section.
If not, create it with the Problem Restatement template.

### 5. FINDINGS
The selected working framing.

### 6. HANDOFF
Pass to KAC Quick or next lean technique.

---

## Watch-Outs
- Don't spend more than 5 minutes — this is a speed check, not deep analysis
- Ensure each restatement genuinely shifts perspective (not just rephrasing)
