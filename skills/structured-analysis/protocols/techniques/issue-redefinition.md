# Protocol: Issue Redefinition

> **Phase**: Launch | **Effort**: Minutes | **Bias Mitigated**: Anchoring
> **Output Artifact**: `working/problem-framing.md`
> **Library**: 00-prime §4 (Essential 8), 01-tradecraft-primer, 09-core-techniques

---

## Purpose

Break anchoring on the initial problem framing. Force consideration of alternative ways to define the central problem before analysis begins.

---

## Execution

### 1. SETUP
- Read Customer Checklist output (`working/requirements.md`) if available, otherwise read conversation context
- Identify the original problem statement as stated

### 2. PRIME
State: "I'll now reframe the problem multiple ways to ensure we're not anchored on the initial framing."

### 3. EXECUTE

1. **State the original problem** exactly as given
2. **Generate 3+ reformulations** using these specific shift types:
   - **Actor to System**: Who is doing this? → What system enables/causes this?
   - **Threat to Vulnerability**: What threatens us? → What vulnerability are we exposed to?
   - **Short-term to Long-term**: What's happening now? → What trend is this part of?
   - **"Why" to "How"**: Why is this happening? → How could this change?
   - **180-degree reversal**: Turn the question completely around
   - **Scope shift**: Expand or contract the problem boundary
3. **For each restatement**: explain what new analytical perspectives it opens
4. **Select the best framing** with explicit rationale — which opens the most productive analytical paths?

### 4. ARTIFACT
Write `working/problem-framing.md` using the Issue Redefinition template.

### 5. FINDINGS
The reframed problem statement and rationale for selection.

### 6. HANDOFF
Pass reframed problem to KAC or the next selected technique.

---

## Watch-Outs
- **Anchoring**: The first framing feels "obviously right" — resist this
- **Superficial restatements**: "How can we fix X?" → "How can we solve X?" is NOT a real reframe. Each must shift perspective meaningfully.
- **Analysis paralysis**: Select one framing and move forward. The goal is better framing, not perfect framing.
