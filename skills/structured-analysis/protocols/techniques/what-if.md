# Protocol: What If? Analysis

> **Phase**: Challenge & Foresight | **Effort**: Hours | **Bias Mitigated**: Status quo bias, Failure of imagination
> **Output Artifact**: `working/what-if.md`

---

## Purpose

Assume a high-impact event has already occurred and work backward to explain how it could have come about. Shifts the analytic question from "whether" to "how," freeing the analyst from probability debates and exposing blind spots in the prevailing view.

---

## Execution

### 1. SETUP
- Read current analytical findings from prior technique outputs
- Identify the prevailing analytic line — the consensus view of what will or will not happen
- Gather relevant evidence from `working/evidence-registry.md`

### 2. PRIME
State: "I will now assume a high-impact event has occurred and explain how it could happen — working backward from the event to the present."

### 3. EXECUTE

1. **State the conventional analytic line** — write out the prevailing view clearly and completely
2. **Define the "What If?" event** — select a specific, high-impact event that challenges the conventional line. The event should be:
   - Consequential enough to matter
   - Not already widely expected
   - Specific enough to reason about (not vague)
3. **Select triggering events** — identify 2-5 plausible catalysts that could set the chain in motion
4. **Think backwards** — develop the chain of argumentation from the assumed event back to present conditions:
   - Start at the event and ask: "For this to have happened, what must have been true immediately before?"
   - Repeat, stepping backward through each causal link until reaching present-day conditions
   - Each link must be logically defensible and supported by evidence where possible
5. **Assess plausibility** — evaluate the overall pathway:
   - How strong is each link?
   - Where is the weakest point?
   - Is there historical precedent for similar chains?
6. **Generate indicator lists** — identify observable events or conditions that would signal the pathway is activating
7. **Consider scope of consequences** — if the event occurred, what domains would be affected and how severely?

### 4. ARTIFACT
Write `working/what-if.md` using the What If? Analysis template.

### 5. FINDINGS
Summarize: the plausibility of the scenario, the key indicators to watch, and the most significant consequences if the event occurred.

### 6. HANDOFF
Pass findings to Premortem for self-critique, or to the next selected technique.

---

## Watch-Outs
- **Status quo bias**: The conventional line will feel "obviously correct." This is precisely the bias this technique counters — push past it.
- **Failure of imagination**: Do not limit the "What If?" event to minor variations of the status quo. The value is in exploring genuinely surprising outcomes.
- **Probability trap**: Do NOT debate whether the event is "likely." The technique assumes it happened. Focus entirely on HOW.
- **Weak chains**: Every link in the backward chain must be defensible. A chain with one implausible link is a weak chain. Strengthen or acknowledge.
